The provided sequence XML is Rabimodulated.xml with mw_freq swept. The active instructions first polarize the NV and take a detection readout labeled in the XML as the true 0 level reference. The optional 1 level reference block is disabled because full_expt = 0, so it is not an active readout. The second active detection follows rabi_pulse_mod_wait_time with length_rabi_pulse = 52 ns and mod_depth = 1.

Using the supplied setup facts, mod_depth = 1 gives about a 10 MHz Rabi frequency, so a 52 ns pulse is approximately a pi pulse. Therefore, on resonance the post-pulse readout should drop strongly relative to the bright reference, with an expected scale comparable to the stated 22 percent mS = 0 to mS = +1 contrast.

The data match that expectation. The first readout is mostly flat near 38 to 40, while the post-pulse readout has a pronounced dip near the middle of the sweep, falling to about 28.8 while the reference is about 38.5. That is roughly a 25 percent drop at the deepest point, close to the expected contrast scale for a resonant pi-scale pulse. The per-average traces show the same central depression, although the stored averages should not be overinterpreted as independent repeatability. This is consistent with a pODMR resonance being present.

The active sequence is Rabimodulated.xml, scanned over mw_freq from 3.825 to 3.925 GHz. The XML first polarizes and detects a true mS = 0 reference, then because full_expt = 0 the optional mS = 1 reference branch is not executed. The sequence then applies rabi_pulse_mod_wait_time with length_rabi_pulse = 52 ns and mod_depth = 1, followed by the signal detection. Thus readout 1 is the pre-microwave reference and readout 2 is the post-pulse signal readout.

At mod_depth = 1 the setup Rabi frequency is about 10 MHz, so a 52 ns pulse is near a pi pulse on resonance. With an mS = 0 to mS = +1 contrast scale around 22%, a real resonance should produce a clear drop of the post-pulse signal relative to the reference, not just a shared drift in both readouts.

The raw readouts both trend downward across the scan, which is consistent with tracking or brightness drift. Comparing readout 2 to readout 1, the ratio fluctuates around unity with only small, inconsistent deviations; there is no stable resonance-shaped depression that stands out from the two stored averages. The stored averages also follow the same broad brightness behavior rather than providing an independent repeatability test. I therefore classify this scan as resonance absent.

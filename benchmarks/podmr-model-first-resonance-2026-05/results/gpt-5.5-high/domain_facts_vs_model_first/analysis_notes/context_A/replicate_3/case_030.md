The provided standalone sequence XML is Rabimodulated.xml with mw_freq swept from 3.825 GHz to 3.925 GHz. The active instructions first polarize and detect, giving a true bright m_S = 0 reference readout. The full_expt variable is 0, so the optional separate m_S = 1 reference block is skipped. The experiment then applies rabi_pulse_mod_wait_time with length_rabi_pulse = 52 ns and mod_depth = 1, followed by the second detection. Thus readout 1 is the bright reference and readout 2 is the post-microwave-pulse signal.

At mod_depth = 1, the setup Rabi frequency is about 10 MHz, so a 52 ns pulse is very close to a pi pulse. If the microwave frequency is resonant, readout 2 should drop by roughly the available m_S = 0 to m_S = +1 contrast scale, about 22%, relative to readout 1.

The combined data show readout 2 falling from the mid-40s to about 35.9 near 3.875 GHz while readout 1 remains near 46.2 there. That is a drop of about 10.3 counts, or roughly 22% of the bright readout, matching the expected contrast for a near-pi pulse. The per-average traces also show the same localized reduction in the post-pulse readout around the same frequency, although the stored averages should not be overinterpreted as a strong independent repeatability test. The feature is therefore consistent with a pODMR resonance.

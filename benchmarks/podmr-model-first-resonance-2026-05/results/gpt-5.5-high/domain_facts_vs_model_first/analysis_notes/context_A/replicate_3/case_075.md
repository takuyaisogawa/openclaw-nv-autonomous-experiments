The provided sequence XML is Rabimodulated.xml with mw_freq as the swept parameter. The active instructions first polarize and detect a true m_S = 0 reference, then skip the m_S = +1 reference branch because full_expt = 0, then apply rabi_pulse_mod_wait_time followed by the second detection. Thus readout 1 is the polarized no-MW reference and readout 2 is the MW-pulse signal readout.

The relevant pulse settings in the provided XML are mod_depth = 1 and length_rabi_pulse = 52 ns. With the stated setup Rabi frequency of about 10 MHz at mod_depth = 1, this is close to a pi pulse. If a resonance were being driven cleanly, the signal readout should show a large, frequency-localized reduction relative to the reference, on the order of the setup contrast scale for a single NV center.

The measured readouts do not show that behavior. The largest readout-2 depressions relative to readout 1 are isolated points around 3.83, 3.845, and 3.88 GHz, with the 3.88 GHz point only about 7.5% below the reference. Other nearby points return to baseline, and the per-average traces show substantial scatter consistent with tracking/noise rather than a stable resonance line. Because the expected near-pi-pulse contrast is much larger than the observed isolated dips, I classify this case as resonance absent.

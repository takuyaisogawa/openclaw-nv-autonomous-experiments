The provided sequence is Rabimodulated.xml with mw_freq as the swept parameter. With full_expt = 0, the "Acquire 1 level reference" block is inactive, so each point contains a first detection after laser polarization as the mS = 0 reference readout, followed by a Rabi-modulated microwave pulse and a second detection as the signal readout.

The active microwave pulse is rabi_pulse_mod_wait_time(PSeq, sample_rate, length_rabi_pulse, mod_depth, switch_delay, ch_on). The provided XML/variable values give length_rabi_pulse = 52 ns and mod_depth = 1. Given the stated setup scale of about 10 MHz Rabi frequency at mod_depth = 1, this is near a pi pulse, so an on-resonance point should transfer substantial population and produce a readout drop approaching the available mS = 0 to mS = +1 contrast scale of about 22%, allowing for experimental imperfections.

The combined signal/reference contrast only reaches about 5 to 6% at its largest negative excursions and changes sign at several adjacent points. There is a small cluster where the post-pulse readout is lower than the reference near 3.88 to 3.895 GHz, plus another point near 3.91 GHz, but these features are weak relative to the expected near-pi contrast and are not cleanly isolated from reference drift/tracking changes. The per-average traces show similar tracking-level structure rather than a strong repeatable resonance signature, and the two stored averages should not be treated as a robust independent repeatability test.

Decision: resonance absent. The data do not show a convincing pODMR resonance for this sequence and pulse setting.

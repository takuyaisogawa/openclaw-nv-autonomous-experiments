The active sequence is Rabimodulated.xml with mw_freq swept from 3.825 GHz to 3.925 GHz in 5 MHz steps. In the sequence, the first detection follows optical polarization only, so readout 1 is the true m_S = 0 reference. full_expt is 0, so the optional m_S = +1 reference block is skipped. The second active detection follows a rabi_pulse_mod_wait_time pulse, so readout 2 is the post-MW measurement used to test for pODMR contrast.

The relevant pulse settings are mod_depth = 1 and length_rabi_pulse = 52 ns. With the stated setup calibration, mod_depth = 1 corresponds to roughly 10 MHz Rabi frequency, so 52 ns is approximately a pi pulse on resonance. A real resonance sampled by this scan should therefore cause a substantial depletion of readout 2 relative to the m_S = 0 reference, on the order of the setup contrast scale rather than only a few percent.

The combined data do not show such a feature. The readout 2 / readout 1 ratios fluctuate around unity, with the deepest dips only about 3-4 percent below the reference and with comparable positive excursions elsewhere. The per-average traces mostly reflect large baseline/tracking offsets between averages rather than a stable resonance feature, and stored averages are not a strong independent repeatability test here. Because the expected on-resonance response for this pulse should be much larger than the observed fluctuations, I classify this case as resonance absent.

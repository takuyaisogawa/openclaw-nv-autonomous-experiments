The sequence is Rabimodulated.xml with mw_freq swept from 3.825 to 3.925 GHz in 5 MHz steps. The active instructions acquire a true m_S = 0 reference first via polarize then detection, then, because full_expt = 0, skip the separate m_S = 1 reference branch and run one rabi_pulse_mod_wait_time followed by the second detection. Therefore readout 1 is the pre-microwave bright reference and readout 2 is the post-pulse measurement.

The relevant pulse is length_rabi_pulse = 52 ns with mod_depth = 1. Using the provided setup fact of about 10 MHz Rabi frequency at mod_depth = 1, this is close to a pi pulse on resonance, so a true pODMR resonance should produce a pronounced contrast-scale reduction in readout 2 relative to readout 1, potentially on the order of the 22% m_S = 0 to m_S = +1 contrast scale, allowing for imperfect transfer and noise.

The raw traces do not show a localized, reproducible resonance-shaped depletion of readout 2 relative to readout 1 across the microwave-frequency sweep. The two combined readouts cross repeatedly and differ by only a few counts with no stable dip at one scan value. The per-average overlay shows substantial tracking/average-to-average variation, and the stored two averages should not be treated as strong independent repeatability evidence. Because the expected on-resonance pi-pulse signature is not clearly present, I classify this case as resonance absent.

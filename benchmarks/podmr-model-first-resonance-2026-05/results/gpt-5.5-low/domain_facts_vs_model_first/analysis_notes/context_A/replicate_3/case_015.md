The instruction order first polarizes and detects the true m_S = 0 reference, then skips the optional m_S = +1 reference because full_expt = 0, then applies rabi_pulse_mod_wait_time using length_rabi_pulse = 52 ns and mod_depth = 1 before the second detection. At the stated setup scale, mod_depth = 1 gives about 10 MHz Rabi frequency, so a 52 ns pulse is close to a pi pulse. If the swept microwave is resonant, the second readout should drop toward the m_S = +1 brightness while the first readout remains comparatively flat. This is roughly a 25% drop from the local bright level, consistent with the stated 22% contrast scale for a near-pi pulse. Both stored averages show the same central depression, although the averages are mainly tracking-cadence checks rather than a strong independent repeatability test.

The active sequence is Rabimodulated.xml with mw_freq swept from 3.825 to 3.925 GHz in 5 MHz steps. The instruction block first polarizes and detects a true m_S = 0 reference, then because full_expt = 0 it skips the separate m_S = 1 reference block, applies rabi_pulse_mod_wait_time with length_rabi_pulse = 52 ns and mod_depth = 1, and detects the post-pulse signal readout. Thus readout 1 is the 0-reference and readout 2 is the microwave-pulse signal.

Using the provided setup facts, mod_depth = 1 corresponds to roughly 10 MHz Rabi frequency, so a 52 ns pulse is near a pi pulse on resonance. If a real pODMR resonance were present, the post-pulse readout should show a fairly strong fluorescence reduction relative to the 0-reference, on the order of the setup contrast scale between m_S = 0 and m_S = +1, about 22% for a good inversion. The observed readouts stay near 50 counts with point-to-point fluctuations of only a few counts and no coherent, localized signal dip of the expected size. The largest apparent deficits are small and not cleanly separated from baseline/tracking variation, while some frequencies show the pulse readout above the reference.

Stored averages are only two and can reflect tracking cadence, so the per-average overlay does not provide strong independent repeatability. Overall the data do not show a credible pODMR resonance.

I inspected inputs/sequence.xml before deciding. The active pulse pattern is the Rabimodulated pODMR sequence while sweeping mw_freq. The sequence first optically polarizes and detects a true 0-level reference. Because full_expt is 0, the optional 1-level reference branch is not active. It then applies rabi_pulse_mod_wait_time with length_rabi_pulse = 5.2e-08 s and mod_depth = 1, followed by the signal detection. Thus readout 1 is the optically prepared 0-level reference and readout 2 is the post-microwave-pulse pODMR signal.

The post-pulse signal readout shows a clear, localized drop around 3.875e9 to 3.885e9 Hz, reaching about 40.5-41.9 raw counts compared with surrounding values mostly in the mid-40s. The same dip is visible in both averages, while the reference readout is comparatively flatter and does not explain the full signal decrease. This is consistent with a pODMR resonance being present.
